   one-liner at the bottom of this file for that PowerShell window only, then whoami again.


--- FIX 2: Personal Access Token (no eas login password at all) ---

0) Clear bad environment variables (do this if you see warnings about company-root.pem
   or NODE_TLS_REJECT_UNAUTHORIZED). You must NOT keep example paths from docs:

   Remove-Item Env:NODE_EXTRA_CA_CERTS -ErrorAction SilentlyContinue
   Remove-Item Env:NODE_TLS_REJECT_UNAUTHORIZED -ErrorAction SilentlyContinue

   If those were set in Windows "Environment variables" (User or System), remove them there too:
   Win+R → sysdm.cpl → Advanced → Environment variables → delete NODE_EXTRA_CA_CERTS and
   NODE_TLS_REJECT_UNAUTHORIZED unless IT gave you a real .pem path.

1) In normal Chrome, open: https://expo.dev/accounts/[your-account]/settings/access-tokens
   (Or: expo.dev → avatar → Account settings → Access tokens → Create token)

2) PowerShell — paste when prompted (avoids copying placeholder text by mistake):

   cd C:\Users\yerram\Desktop\HDFC\mobile
   $env:EXPO_TOKEN = Read-Host "Paste Expo access token from website"
   npx eas-cli@latest whoami

   If whoami says "bearer token is invalid", create a NEW token on expo.dev (old one may be revoked) and run Read-Host again.

3) If whoami prints your account, skip "eas login" forever for this session. Run:
   npx eas-cli init
   npm run eas:apk-link

4) Close PowerShell when done (clears EXPO_TOKEN). Next time, set EXPO_TOKEN again or use login --browser.


--- Normal flow (only if login works) ---

1) Open PowerShell or CMD and go to the mobile folder:
   cd C:\Users\yerram\Desktop\HDFC\mobile

2) Log in to Expo:
   npx eas-cli login
   (Prefer: npx eas-cli login --browser)

3) Link this folder to an Expo project (first time only — follow prompts):
   npx eas-cli init

4) Start the Android build and WAIT until it finishes (then copy the https://expo.dev/artifacts/eas/....apk line):
   npm run eas:apk-link

   If the command ends before the build is done, open the "Build details" URL it printed,
   wait for "Finished", then click Download and copy the .apk URL from the browser.

Your site URL is already set for the "preview" build in eas.json (EXPO_PUBLIC_WEB_APP_URL).
To use a different URL later, edit mobile/eas.json → build.preview.env.
